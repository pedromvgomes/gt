## Highlights

- **SSH origin rewriting now falls back to `~/.ssh/config`.** When you clone
  `git@github.com:owner/repo.git` and gt has no matching `ssh.host_aliases`,
  `gt clone` (and `gt set-ssh-remote`) parse `~/.ssh/config` for `Host` blocks
  whose `Hostname` matches the canonical host and use them — silently for a
  single match, with an interactive menu when multiple identities are
  configured. SSH-input URLs now go through the alias resolver too; previously
  only HTTPS inputs were rewritten.
- **Agentic skill: `use-gt`.** Adds `agentic/skills/use-gt/` so Claude Code (and
  similar coding agents) drive gt for clones, worktrees, setup, and auth
  instead of raw `git`, plus a short rules-of-the-road doc on the
  worktree-per-session layout.

## SSH-config fallback

The end state we wanted: clone a repo with an SSH URL on a machine that
already has the right key configured under a `~/.ssh/config` `Host` alias,
and pushes Just Work without a manual `git remote set-url`.

`gt`'s alias resolver now layers like this for a host like `github.com`:

1. `ssh.user_aliases[<--user>][github.com]` (per-user override)
2. `ssh.host_aliases[github.com]`
3. Matching non-wildcard `Host` blocks in `~/.ssh/config` whose
   `Hostname` resolves to `github.com`:
   - **0 matches** → URL kept as-is.
   - **1 match** → used silently with an `INFO` log.
   - **2+ matches, interactive** → menu with a numbered list and a
     `keep <host> (no rewrite)` escape, plus a hint showing the YAML to
     persist the choice in `~/.config/gt/config.yaml`.
   - **2+ matches, non-interactive** → URL kept, warning emitted.

Already-aliased URLs (`git@github-personal:owner/repo.git`) are detected and
not double-rewritten.

`gt set-ssh-remote` benefits automatically — useful for repos cloned before
this release where the origin still points at the canonical host.

## Upgrading

```sh
gt update
```

No config migration needed. To opt in to silent rewrites without prompts,
add the relevant entries to `~/.config/gt/config.yaml`:

```yaml
ssh:
  host_aliases:
    github.com: github-personal
  # or, for multiple identities:
  user_aliases:
    pedromvgomes:
      github.com: github-personal
    pedro-work:
      github.com: github-work
```
